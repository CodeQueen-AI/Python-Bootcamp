**Definition:**  
Functions in Python are reusable blocks of code that perform a specific task. They help organize programs and avoid repetition.


## 1. Basic Functions
- Defined using the `def` keyword.  
- Can be called multiple times to perform the same task.  



## 2. Functions with Parameters
- Functions can accept inputs (parameters) to work with dynamic values.  



## 3. Functions with Default Arguments
- Parameters can have default values, used if no value is provided.  



## 4. Function Return Values
- **No return** â†’ Executes code but returns `None`.  
- **Single return** â†’ Returns one value.  
- **Multiple return** â†’ Can return multiple values at once.  
- **Return list/dictionary** â†’ Functions can return collections.  


## 5. Higher-Order Functions
- Functions that can take other functions as arguments.  
- Functions can return another function.  
- Lambda functions can also be returned.  



## 6. Positional, Keyword, and Arbitrary Arguments
- **Positional arguments** â†’ Passed in order.  
- **Keyword arguments** â†’ Passed with names (`name="CodeQueen"`).  
- **Arbitrary positional arguments (*args)** â†’ Accept multiple values.  
- **Arbitrary keyword arguments (**kwargs)** â†’ Accept key-value pairs.  


## 7. Lambda (Anonymous) Functions
- Small, unnamed functions written using `lambda`.  
- Can have single or multiple parameters.  
- Often used with `map()`, `filter()`, and `reduce()`.  



## 8. Function Scope
- **Local scope** â†’ Variables inside a function are only accessible there.  
- **Block scope** â†’ Python does not strictly enforce block scope, but variables inside blocks are still accessible outside.  


## 9. Built-in Functions with Functions
- **reduce()** â†’ Repeatedly applies a function to a sequence.  
- **zip() / unzip** â†’ Combines or separates iterables.  
- **sorted()** â†’ Returns a sorted list.  
- **enumerate()** â†’ Adds index while looping.  



## 10. Special Functions (Object-Oriented)
- **Constructor (`__init__`)** â†’ Used to initialize an object.  
- **`__str__` method** â†’ Defines how an object is represented as a string.  
- **`__len__` & `__getitem__`** â†’ Special methods to make objects behave like built-in data structures.
